# Android app architecture foundations

This sequence is intended for Android developers who know the basic platform
APIs and want a structured introduction to application architecture. Its goal
is to connect platform constraints, architectural boundaries, state and data
principles, UI and data-layer design, dependency management, and application
composition.

The package references canonical competency set `android-app-architecture`
version `2`. It starts with Android lifecycle constraints, establishes general
boundaries and state principles, applies them to UI and data layers, and leaves
conditional application-scale decisions until the final stage.

Stage order is a recommended pedagogical order for this sequence, not a
mandatory dependency or a canonical relation between competencies.
Competencies within the same stage are grouped as a coherent instructional unit
and have no semantic order.

The sequence is version `1` and remains in `review`; it is not approved or
published.
