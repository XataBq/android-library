## Educational objectives

The topic introduces the conceptual foundations of modern Android application architecture.

It explains:

- why architecture exists;
- separation of concerns;
- Android component lifecycle limitations;
- UI layer;
- Data layer;
- optional Domain layer;
- dependency direction;
- Single Source of Truth;
- Unidirectional Data Flow;
- architectural responsibility boundaries.

The topic intentionally avoids implementation details.

---

## Learning outcomes

The learner should be able to:

- explain why Android framework components should not own application data;
- distinguish UI, Data and optional Domain layers;
- explain separation of concerns;
- explain dependency direction;
- explain SSOT;
- explain UDF;
- identify responsibility violations;
- justify architectural decisions.

Learning outcomes must use observable verbs only.
