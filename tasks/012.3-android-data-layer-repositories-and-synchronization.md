# Task 012.3 --- Android Data Layer, Repositories, and Synchronization

Status: DONE

## Objective

Create the repository's third production educational topic package.

The topic must teach how the Android Data layer provides application
data through repositories, coordinates local and remote data sources,
defines a Single Source of Truth (SSOT), handles caching and
synchronization, and isolates the rest of the application from storage
and transport details.

The topic builds on:

-   Task 012.1 --- Android App Architecture Foundations
-   Task 012.2 --- Android UI Layer and Unidirectional Data Flow

## Required conceptual model

``` text
UI / Domain
     ↓
 Repository
     ↓
Data source coordination
 ↙               ↘
Local          Remote
     ↓
Application data
```

Also explain local-first synchronization:

``` text
Remote
   ↓
Persist locally
   ↓
Observe local storage
   ↓
Expose updated application data
```

Clarify that these are conceptual responsibility flows, not mandatory
class diagrams.

## Learning outcomes

The learner should be able to:

-   explain Data layer responsibilities;
-   distinguish repositories from data sources;
-   identify the SSOT;

-   compare cache-first, network-first and local-first strategies;
-   explain synchronization trade-offs;
