# Project Vision

## Product

`android-library` is a long-term competency-driven learning platform for Android and software engineering.

It combines:

- a structured knowledge base;
- a dependency-based learning roadmap;
- practical programming tasks;
- interview preparation;
- testing and assessment;
- spaced repetition;
- progress tracking;
- AI-assisted tutoring;
- web and Android clients.

## Mission

Create one maintainable system in which a developer can learn, practise, verify, retain, and revisit engineering knowledge without relying on scattered notes and random learning sources.

## Primary goals

### Educational goal

Support structured growth through engineering competencies, initially focused on:

- Java;
- Kotlin;
- JVM;
- computer science;
- algorithms and data structures;
- concurrency and coroutines;
- networking;
- architecture;
- Android framework;
- testing;
- security;
- system design.

### Engineering goal

Develop a real product that demonstrates:

- domain modelling;
- modular architecture;
- content modelling;
- validation and automation;
- web development;
- Android development;
- backend development;
- offline-first design;
- CI/CD;
- testing;
- AI integration.

### Product goal

Begin as a personal learning system and remain capable of evolving into a platform usable by other developers.

## Success criteria

The system should eventually allow a learner to:

1. discover what to study next;
2. understand topic prerequisites;
3. read structured theory;
4. use a concise cheat sheet;
5. complete tests and practice;
6. train interview answers;
7. track mastery and weak areas;
8. receive scheduled reviews;
9. access content from GitHub, web, and Android;
10. use AI for explanation and assessment without replacing active learning.

## Non-goals for the initial phases

The project will not initially:

- support multiple users;
- contain a custom backend;
- provide social features;
- provide payments or subscriptions;
- generate all content automatically;
- attempt to replace a full LMS;
- implement every future module before real usage validates it.
